# Apps Script: 페어링 박제 자동화

## 목적

Google Sheets의 `RAND()` 휘발성 함수가 매번 재계산되어 시트 화면과 대시보드가 어긋나는 문제를 해결합니다. 이 Apps Script는 시트의 메뉴에 박제 버튼을 추가해서 클릭 한 번으로 RAND() 수식을 정적 값으로 변환합니다.

## 설치 방법 (대회별 시트마다 1회)

대회 001, 002, 003 각 시트에 동일하게 설치합니다.

### 1. 대상 시트 열기
예: [잭앤질-001](https://docs.google.com/spreadsheets/d/1gzX4kidjg4J6Qj5g1ANX9ibdeGaK_KkLTgU6xoQVn80)

### 2. Apps Script 편집기 열기
시트 메뉴 → **확장 프로그램** → **Apps Script**

### 3. 코드 붙여넣기
- 새 탭에서 `Code.gs` 기본 내용 모두 삭제
- [freeze-pairing.gs](./freeze-pairing.gs) 내용 전체 복사 → 붙여넣기
- `Ctrl+S` 저장
- 프로젝트 이름 입력 (예: "JNJ Pairing Freeze")
- 첫 실행 시 권한 요청 → "허용" 클릭

### 4. 시트 새로고침
시트 탭으로 돌아가서 **F5** → 메뉴에 **🎲 페어링 관리** 추가됨

## 사용 방법

### 일상 운영 흐름

```
대회 시작 직전
    ↓
시트 → 🎲 페어링 관리 → "셔플 후 박제 (예선)"  ◀ 클릭 한 번
    ↓
RAND() 자동 재계산 → 정적 값으로 박제 완료
    ↓
대시보드 → ↻ 시트에서 페어링 락 클릭
    ↓
시트와 대시보드 영구 일치 ✅
```

### 메뉴 항목 설명

| 메뉴 | 설명 |
|---|---|
| **현재 페어링 박제 (예선)** | 지금 보이는 예선 페어링을 정적 값으로 고정 (셔플 안 함) |
| **현재 페어링 박제 (본선)** | 지금 보이는 본선 페어링을 정적 값으로 고정 |
| **현재 페어링 박제 (전부)** | 예선·본선 한 번에 박제 |
| **셔플 후 박제 (예선)** | J1 증가로 새 셔플 + 즉시 박제 |
| **셔플 후 박제 (본선)** | 본선 동일 |
| **박제 해제 (예선)** | 백업해 둔 RAND() 수식 복원 |
| **박제 해제 (본선)** | 본선 동일 |
| **박제 해제 (전부)** | 예선·본선 모두 수식 복원 |

## 동작 원리

### 박제

1. 대상 범위(`A2:G21` 또는 `A2:G11`)의 모든 수식을 읽음
2. 현재 계산된 값을 읽음
3. 원본 수식을 숨김 시트 `_pairing_formula_backup`에 JSON으로 저장
4. 범위에 정적 값을 다시 쓰기 → 수식 사라짐, RAND() 영향 없어짐

### 박제 해제

1. `_pairing_formula_backup` 시트에서 백업된 수식 로드
2. 범위에 수식 복원 → RAND() 다시 작동

### 백업 시트

- 이름: `_pairing_formula_backup`
- 자동 숨김 처리됨
- 컬럼: `key | formulas_json | timestamp`
- 박제 후 데이터를 영구 보관 (수동 삭제 안 하면 무한 보관)

## 주의 사항

- **첫 실행 시 권한 요청**: Google이 "검토되지 않은 앱" 경고를 띄울 수 있음 → "고급" → "안전하지 않음으로 이동" → "허용" 진행
- **각 대회별로 따로 설치 필요**: Apps Script는 시트별로 독립적
- **박제 후엔 셔플 불가**: 정적 값이므로 새 셔플하려면 박제 해제 → F9 → 다시 박제

## 트러블슈팅

### 메뉴가 안 보임
- 시트 새로고침(F5) 안 했을 가능성 → F5
- onOpen 트리거 등록 안 됐을 수 있음 → Apps Script 편집기에서 `onOpen` 함수 한 번 수동 실행

### "탭 없음" 토스트
- `PAIRING_TABS` 상수의 탭 이름이 실제 시트 탭 이름과 다름 → 코드에서 수정

### 박제 해제 시 "백업 없음"
- 박제한 적이 없거나 `_pairing_formula_backup` 시트가 삭제됨
- 마스터 템플릿에서 새로 사본 만들어 다시 셋업 필요
